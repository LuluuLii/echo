# Echo 产品哲学

> 从「被打动」到「能表达」的完整旅程

---

## 核心信念

### 语言是思维的边界

你能表达的，才是你真正理解的。
你表达不出的，往往是你还没想清楚的。

**锤炼语言，就是锤炼思维；了解自己的表达，就是了解自我。**

### 表达的核心不是技巧，而是「有话想说」

大多数人表达不好，不是因为词汇量不够，而是因为：

1. **没有被语言打动过** — 不知道好的表达是什么样
2. **没有想要表达的东西** — 肚子里空空，技巧再好也无用

---

## 表达的三层基础

```
        ┌─────────────────┐
        │    技巧层       │  ← 词汇、句式、语法
        ├─────────────────┤
        │    感知层       │  ← 语言之美、表达的力量
        ├─────────────────┤
        │    源泉层       │  ← 态度、观点、真实的生命经验
        └─────────────────┘
```

**传统语言学习**：只关注技巧层 → 学了很多，表达干瘪

**Echo 的方式**：从源泉层开始 → 有话想说，自然会找到表达方式

---

## Echo 的三个目标

| 层次 | 目标 | 如何实现 |
|------|------|---------|
| **感受语言之美** | 被好的表达打动 | 用户收集触动自己的素材 |
| **积累表达的源泉** | 有态度、有观点、有经历 | 素材与个人经历关联，激活卡唤醒真实感受 |
| **练习表达输出** | 把想法说出来 | Echo Session：从感受到表达的完整闭环 |

---

## 为什么从「素材」开始

素材不是学习材料，而是**触动你的瞬间**：

- 一句让你停下来的话
- 一个让你有共鸣的观点
- 一段让你想起自己经历的描述

**收集素材的过程，就是培养语言感知力的过程。**

当你开始注意到好的表达，开始被语言打动，你离好的表达就不远了。

---

## 表达循环

```
被触动（感知语言之美）
    ↓
产生共鸣（与自我经历连接）
    ↓
想要表达（有了态度和观点）
    ↓
说出来（锤炼语言 = 锤炼思维）
    ↓
回看（了解表达 = 了解自我）
    ↓
更敏锐地感知（循环提升）
```

每一次表达，都是一次自我探索。
每一个 Artifact，都是思维的结晶。

## 设计原则

### 只有表达过的内容才会被留下

```
素材积累 → 召回 → 激活 → 表达 → 作品保存
    ↑                ↓              │
    │           (阅后即焚)          │
    └───────────────────────────────┘
```

| 内容类型 | 是否持久化 | 说明 |
|----------|------------|------|
| **素材 (Materials)** | ✅ 保存 | 触动你的片段，表达的原料 |
| **激活卡 (Card)** | ❌ 不保存 | 邀请函，用于唤醒感受，阅后即焚 |
| **表达 (Artifact)** | ✅ 保存 | 你真正说出来的作品 |

**激活卡是镜子**：帮你看到内心，但镜子本身不需要保存。
**表达是证明**：只有说出来、写下来，想法才真正存在。
**作品是自我**：你的 Artifacts 集合，就是语言视角下的「你」。

### 离线优先

核心价值循环在离线时 100% 可用。

AI 是增强，不是依赖。没有网络时，用户依然可以：
- 收集素材
- 浏览回顾
- 自由表达
- 保存作品

### 被动 → 主动 的词汇转化

```
被动词汇（在素材中见过）
    ↓ 激活练习
主动词汇（在表达中用过）
    ↓ 反复使用
掌握词汇（自如运用）
```

追踪「见过什么」vs「用过什么」，推荐「见过但没用过」的词汇。

## 与传统语言学习的区别

| 传统 App | Echo |
|---------|------|
| 学「正确的表达」 | 找到「属于自己的表达」 |
| 模仿他人 | 发现自我 |
| 输入驱动 | 输出驱动 |
| 追求标准答案 | 追求真实表达 |
| 外部教材 | 用户自己的素材 |
| 刷题背单词 | 表达练习 |
| 被动吸收 | 主动输出 |

## 一句话总结

**Echo：帮助用户感受语言之美，积累表达的源泉，在表达中认识自我。**
