# Echo - Every note echoes back
“个人素材激活与人生拼图”工具，它帮助你从碎片化积累中，发现线索、形成连接、激发创作，陪你构建有意识的表达和自我认知。
① 解决“素材闲置”的问题，让用户能在创作或输出时主动调动已有积累。
② 解决“被信息流裹挟”的问题，让用户回到自己的内容，形成一种反思和自我成长的体验。
你希望产品不仅仅是一个“工具”，而是有温度、有陪伴感的产品，带有哲学性，让用户感受到“我是谁，我在构建什么”。

## 原型 demo
```
[Journey] 推送流
 ├─ 今日灵感（聚合关联）e.g. “你 3 月份收藏的文章和昨天的笔记，都提到‘创作焦虑’，要不要结合这两个想法，写点什么？”
 ├─ 主题线索（动态发现）每天 AI 会帮你回顾过去积累的片段，并“串联”它们，形成主题线索
 └─ 灵活触发搜索（用户随时输入目标）

[Creative Studio] “激活素材 → 引导输出”
 ├─ 灵感工作台（任务生成）
 ├─ 目标驱动（写作/播客/小红书）
 └─ 输出模板（卡片、脚本）

[Raw Library 原料库] 浏览和管理导入的 Notes、收藏等。
 ├─ 笔记
 ├─ 文件夹
 ├─ 收藏（后续扩展）
 └─ 主题聚类视图 - 展示主题云 / 时间脉络; 关联建议：点击某个素材时，AI 推荐相关内容（帮助用户看到潜在连接）。

[Insight]
 ├─ 每周回顾
 └─ 人生拼图（可视化成长轨迹/主题网络）像一张动态地图，慢慢显现用户的兴趣、价值观。

```

## 核心模块

### **1. 数据接入层**

- **Notes.app 读取**：通过 macOS/iOS API（`EventKit` / `Notes API`），或通过导出方式读取。
- **指定文件夹扫描**：支持 markdown / txt / pdf。
- **（后续）微信聊天记录导入**：先通过手动导出 CSV / JSON 文件再导入。
- **（后续）相册截图导入**: 支持多模态检索/ RAG
- 数据存储：本地 SQLite / CoreData，初期无需云端。

### 2. 数据处理与 AI 分析层

- **关键词提取 & 聚类**（用于后续卡片生成和内容推荐）
- **Embedding + 向量检索**（可用 OpenAI Embeddings + FAISS）
- **差距分析**（用于 Language Learning）
    - 对比用户素材 vs. 目标词库（如雅思常用词）
- **卡片生成器**：
    - 输入：用户笔记片段 + 场景指令
    - 输出：结构化卡片（标题 + 关键内容 + 交互选项）

### 3.  UI
#### 3.1 Journey（灵感发现）主流程
[Journey 首页]
 ├─ 顶部：今日灵感流（卡片列表）
 │    ├─ 卡片类型：
 │    │    - AI 主题关联卡（基于素材库 & 用户近期关注）
 │    │    - 复盘卡（基于过去收藏/笔记）
 │    │    - 行动提示卡（写一篇短文、录音练习等）
 │    └─ 用户交互：
 │         - 点击卡片 → 详情页（显示关联素材 + AI 生成建议）
 │         - 长按 → 收藏 / 标记灵感
 │         - 右滑 → 生成任务 → [Creative Studio]
 │
 ├─ “探索更多”入口
 │    - 输入目标（如“写 IELTS Essay”）
 │    - AI 搜索相关素材 + 提供创作提示
 │    - 跳转 → [Creative Studio]
 │
 └─ “查看历史回顾” → 跳转 [Insight]

[Journey → Creative Studio]
- 入口：点击灵感卡 / 右滑 → 转为任务
- AI 动作：从卡片主题生成创作计划 + 相关素材拼盘

#### 3.2 Creative Studio
[Creative Studio 首页]
 ├─ 灵感工作台（当前任务列表）
 │    ├─ 来自 Journey 推送 / 用户主动添加
 │    ├─ 每个任务包含：
 │    │    - 目标说明（写作/播客/社交媒体）
 │    │    - AI 提供的素材拼盘（笔记摘录 + 关键词）
 │    │    - AI 生成初稿 / 提示
 │    └─ 用户交互：
 │         - 编辑目标 → AI 更新建议
 │         - 调整素材 → 重新生成
 │         - 完成后 → 保存到输出库
 │
 ├─ “新建任务”
 │    - 用户输入目标
 │    - AI 调用素材库，生成草稿提示
 │
 └─ 输出模板
      - AI 推荐不同结构（短文/脚本/卡片）

#### 3.3 Raw Library 原料库
[Raw Library（原料库）]
   ├─ Notes（Notes.app笔记）-- 导入按钮和操作
   ├─ Files & Documents（导入文件：Markdown / PDF / TXT）-- 导入按钮和操作
   ├─ Collections（收藏夹 / 书签）
   ├─ 全部素材总览：卡片网格布局 + 搜索/筛选
   └─ 可拖拽或右滑 → Creative Studio 使用

#### 3.4 Insight
- **核心交互**：
    - AI 自动生成“主题演化图” → 用户看到自己关注的领域随时间的变化。
    - 每个主题可以点击，查看相关素材、相关输出。
- **呈现方式**：
    - **时序地图**（时间线 + 气泡大小）→ 显示你关注的关键词/主题随时间增长。
    - **关系图**（知识网络）→ 让用户看到自己的“知识宇宙”。
- **哲学性文案**：
    - _“这些是你思想的足迹，一点一滴汇聚成今天的你。”_
    - _“这是你的信息森林，你正在为它浇水。”_

### 4.  关键 Human-in-the-loop 场景
- 卡片生成前：用户选择重点领域（比如 IELTS Writing / 小红书内容）
- 卡片生成后：用户编辑/调整，系统学习偏好


## **技术选型**

- **前端
  - POC 阶段：前端：使用 Web 技术，Typescript + React 技术栈，服务端：Node
  - 正式版：（iOS & Mac 共用）：SwiftUI**
    - 可复用大部分 UI 组件，macOS Catalyst 适配。
- **AI 模块**：OpenAI GPT-4/4-mini / local LLM
- **数据索引**：SQLite + FAISS（本地向量库）
- **后续同步**：CloudKit（iCloud 同步）
- **快捷键支持**：iOS Shortcuts + Share Extension（用于快速添加内容）

## POC 验证
最精简链路：素材录入 → 素材激活 → 灵感生成 → 输出 → 回顾
- 读取手机/mac notes.app、相册、文件夹内容，建立本地素材库链接和索引
- 支持根据用户关键词从互联网检索「与素材库内容相关」的 topic，poc 支持：1.检索相关的雅思作文题目；2.检索相关的小红书热点话题；3.自主推荐（根据素材库随机）
- 推送任务和提示引导词，引导用户根据素材进行输出
- 推送首页卡片：卡片生成
核心验证点：
- 基于素材库，进行 AI 检索和关联话题的能力
- 任务和卡片生成的能力
- 模型 token 消耗量、成本、本地模型的可行性
